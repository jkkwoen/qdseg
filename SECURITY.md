# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | :white_check_mark: |
| < 0.2   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please:

1. **Do NOT** create a public GitHub issue
2. Email the maintainer directly at the contact information in the GitHub profile
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

We will respond to security reports within 48 hours and work on a fix as soon as possible.

## Security Best Practices

When using this package:

- Keep your dependencies up to date
- Use `.env` files for sensitive configuration (never commit them)
- Review the `.gitignore` to ensure sensitive files are excluded
- Use virtual environments to isolate dependencies
